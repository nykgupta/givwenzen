[[!-GivWenZen-!][.FrontPage]] 

The step classes are instantiated by looking for constructor in the following order:

 1 Any custom !-org.givwenzen.annotations.InstantiationStrategy-!
 1 A constructor that takes an instance of the interface implemented by the !-GivWenZenExecutor-!, user's custom state class, or a super class of one of these objects  as a parameter
 1 A constructor that takes an instance of !-GivWenZenExecutor-! or user's custom state class or super class of one of them as a parameter
 1 An no parameter constructor

!5 Custom Instantiation Strategy

To do this you will need to have create an implementation of !-org.givwenzen.annotations.InstantiationStrategy-!.

Implement the method:
{{{
   Object instantiate(Class markedClass, Object parameter)
}}}
If the step Class that is passed in as the first parameter can be created using your custom strategy then create and return the object.  If it cannot be created by your strategy then return null.  (I would like to change this.)
To use this custom implementation you must create the !-GivWenZenExecutor-! with the constructor:
{{{
public GivWenZenExecutor(Object stepState, DomainStepFinder domainStepFinder, DomainStepFactory factory)
}}}
 * If you have no stepState object to pass to your steps then this parameter can be null.
 * If you do not need a custom step package you can use the default constructor for !-DomainStepFinder-!.
 * Create !-DomainStepFactory-! using the constructor that takes 1 or more !-org.givwenzen.annotations.InstantiationStrategy-!.
